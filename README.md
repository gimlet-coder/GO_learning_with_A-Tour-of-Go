## 気がついた注意事項

学習していくうえで気になった注意点を書き記していきます

---

### 1. 同一フォルダ内における main 関数は 1 つのみ

* フォルダの中に main 関数が入ったファイルが複数あるとエラーがでます
    * 対策1: 関数を main から main_1 といったように変更する
        * main 関数が入っていない単体ファイルは実行できないためやや不自由
    * 対策2: フォルダをその都度作成して 1 フォルダ 1 main 関数の原則を守る (こちらを採用)
        * フォルダを毎回作る煩わしさはあるもののそのまま実行できる利便さを優先した


---

### 2. フォーマット機能を使うと自動でインデント及び未使用パッケージ, 変数を削除される

* VSCode を用いている場合, Ctrl + S で保存しようとすると自動でフォーマットされます
    * これにより, 例えば上から実直に import まで書いて一旦保存..とすると import の中身をどこにも使っていないのでまるっと消えてしまいます
        * 対策: main 関数から書いて必要に応じて import をしていく書き方をすると回避可能
            * C++ みたいによく使うから algorithm を入れといて使うかは後で考えようはできない
            * 補足: VSCode では main 関数で import していないパッケージの変数や関数を用いると, フォーマットした際に自動で import 処理をしてくれる (これがかなり便利)
